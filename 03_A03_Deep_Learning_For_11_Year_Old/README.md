# A03 — Deep Learning Explained for an 11-Year-Old

**Course:** ITAI 2376 — Deep Learning Artificial Intelligence
**Author:** Jiri Musil

## Problem Statement

Explaining technical material to a non-expert is one of the best tests of whether you actually understand it. This assignment required me to explain deep learning to an 11-year-old — not by dumbing it down, but by stripping out jargon and finding analogies that connect to a child's experience of the world.

## Approach

- Identify the parts of deep learning that an 11-year-old already understands without realizing it (recognizing a friend in a photo, finishing a sentence, learning to ride a bike).
- Use everyday analogies for technical concepts: the brain has neurons, computers have neurons too; a network "learns" the same way you learn what a dog looks like — by seeing many examples.
- Pick concrete, visible applications that a kid can relate to: photo apps that find faces, voice assistants, video game characters that get smarter, image generators.
- Keep slides visual, low-text, and oriented around the question "what would a kid actually want to know?"

## Learning Outcomes

- The constraint of "explain it to an 11-year-old" forced me to drop every shortcut that depends on math vocabulary. I came out of this assignment with a much better story for what deep learning actually does, which I reused later when I had to explain my final project to non-technical stakeholders at work.
- This was a useful reminder that AI literacy is *not* the same as AI competence. You can build the model and still not be able to explain it. Both skills matter.
